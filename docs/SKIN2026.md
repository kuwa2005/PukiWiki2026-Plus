# SKIN2026 — Plus モダンスキン（React）

`pukiwiki/skin2026/` は Plus 向け UI スキンです。標準 `pukiwiki/skin/` は変更せず ini で切り替えます。

**2026 版:** Vite + React 18 のアプリシェルが PukiWiki のサーバー描画 HTML（`$body` / `$menu` 等）を DOM 移譲で再配置し、Notion / Linear 風の contemporary web app 体験を提供します。

## 有効化

`pukiwiki/pukiwiki.ini.php`（git 除外）:

```php
define('SKIN_DIR', 'pukiwiki/skin2026/');
define('SKIN_FILE', DATA_HOME . 'skin2026/pukiwiki.skin.php');
```

雛形: `pukiwiki/pukiwiki.ini.php.example`

## アーキテクチャ

```
pukiwiki.skin.php (PHP)
  ├─ 非表示 #s26-ssr に PukiWiki 出力（#body, #menubar, h1.title …）
  ├─ #s26-config JSON（ナビ・リンク・ページ状態）
  └─ dist/skin2026-app.js（React IIFE, flushSync 同期マウント）

React App (src/)
  ├─ サイドバー（折りたたみ + menu プラグイン）
  ├─ トップバー + ヒーロー（タイトル / topicpath）
  ├─ コマンドパレット（⌘K / Ctrl+K）
  ├─ モバイルボトムナビ + 編集 FAB
  └─ useLayoutEffect で SSR ノードをスロットへ移譲（ID 維持）

PukiWiki JS（変更なし、skin2026 内コピー）
  main.js / search2.js / ref-popup.js / edit-dragdrop.js
```

### PukiWiki 互換の要点

| 要素 | 用途 |
|------|------|
| `#body` | 本文・編集フォーム。main.js / プラグイン |
| `h1.title` | topicpath 書き換え（main.js） |
| `#attach` / `#toolbar` | edit-dragdrop.js |
| `#pukiwiki-site-properties` | `$html_scripting_data`（Core 出力） |

React は **HTML を再生成せず** 既存ノードを移動するため、フォーム・プラグイン出力・#ref 画像はそのまま動作します。

## ビルド

```bash
cd pukiwiki/skin2026
npm install
npm run build   # → dist/skin2026-app.js, dist/skin2026-app.css
```

`dist/` をコミットに含めます（PHP サーバーから CDN ビルド不要で配信）。

開発時:

```bash
npm run dev     # Vite dev server（PHP 連携は手動）
```

## デザイン

- グラスモーフィズム + グラデーションオーブ背景
- ダーク / ライト（`localStorage: pukiwiki-skin2026-theme`）

- Wiki 本文 typography: `pukiwiki.css`（プラグイン・編集 UI）

## デプロイ

上書きデプロイで `skin2026/`（**`dist/` 含む**）を本番へコピー。[UPGRADE.md](UPGRADE.md)  
本番 `pukiwiki.ini.php` は上書きしないため、初回は skin2026 設定を人手で追加。

## 制限・既知事項

- **ビルド必須:** `npm run build` 後の `dist/` が無いと React シェルは表示されない（本文 SSR は `#s26-ssr` に残るが非表示）。
- **toolbar:** 旧来の画像ツールバーは DOM 互換用に `#toolbar` を hidden 保持。UI は React ナビ / FAB / コマンドパレットに集約。
- **topicpath:** `SKIN_DEFAULT_DISABLE_TOPICPATH` 既定 0（topicpath 表示）。Core main.js が `h1.title` を書き換える挙動は維持。
- **skin2026.js:** 旧 vanilla UI ヘルパー。React 版では未使用（テーマは React 側）。

## 開発

変更は Plus リポジトリの `pukiwiki/skin2026/` のみ。Core (`D:\00_project\pukiwiki2026`) には触れない。

本体（`lib/`・`plugin/`・`index.php` 等）の改変が必要になった場合は [CORE-BOUNDARY.md](CORE-BOUNDARY.md) §6 に従い、実装前にユーザー判断を得る。

### 要ユーザー判断（本体依存の可能性）

以下は **skin のみでは未確定** の項目。実装・検証時に問題が出たら [CORE-BOUNDARY.md](CORE-BOUNDARY.md) §6 の A/B/C をユーザーに提示すること。**本体は黙って改変しない。**

| 項目 | 現状・懸念 | 想定される選択肢 |
|------|------------|------------------|
| **`edit-dragdrop.js` と `#toolbar`** | React シェルは `#toolbar` を非表示化。編集時の D&D UI がツールバー前提の箇所あり | **A**: skin 内 JS で別スロットへ誘導／**B**: 本体の編集 UI フック変更 |
| **`$head_tag` / プラグイン注入スクリプト** | プラグインが旧 DOM 構造を前提にしている場合、React マウント後に動作しない可能性 | **A**: skin 側でマウント順・スロット調整／**B**: 該当 plugin 改変／**C**: Core handoff |
| **コマンドパレットのインライン検索** | 現案は静的ナビ＋検索ページへのリンク。Wiki 全文をクライアント検索する API は本体にない | **A**: 検索ページリンクのみ／**C**: 検索 API を Core に追加依頼 |
